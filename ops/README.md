# Workspace Ops System

This `ops/` directory is your command center for building interns and steering all projects.

## Structure

- `00-dashboard/` - active command center views (today, this week, blockers)
- `01-direction/` - vision, constraints, quarterly direction
- `02-interns/` - hiring, onboarding, scorecards, SOPs
- `03-systems/` - recurring processes and automations
- `04-experiments/` - tests and prototypes
- `05-decisions/` - decision logs and rationale
- `06-reviews/` - weekly/monthly retrospectives
- `07-prompts/` - prompt templates for thinking and delegation
- `08-assets/` - reusable docs, snippets, checklists
- `09-archive/` - closed cycles

## Operating Rhythm

1. Start day in `00-dashboard/daily/`.
2. Plan week in `00-dashboard/weekly/`.
3. Run intern pipeline from `02-interns/`.
4. Record non-trivial decisions in `05-decisions/`.
5. Close week with `06-reviews/weekly-review-template.md`.

## Rule

If it's strategy/ops/thinking, it goes in `ops/`.
If it's product/code execution, it goes in `projects/*`.
